# Factor Notes — 工程笔记与因子日记

## 目标

把因子核对、策略调优、代码重构过程中产生的**碎片化高价值信息**固化为可检索的 markdown 文档，防止遗忘、便于后续 agent 快速了解历史决策。

## 触发条件

- "记一下"
- "记录下来"
- "写进 notes"
- "这个因子以后要注意"
- "后续可以考虑..."
- "这里有个坑，记着"
- 任何需要持久化记忆但不需要写完整日报的场景

## 目录结构

```
/home/notes/
├── README.md           # 目录说明与写作规范
├── factor/             # 因子质量评估、因子调优
├── infra/              # 数据流水线、基础设施
├── model/              # 模型训练、特征选择
└── misc/               # 其他杂项
```

## 写作规范

### 文件名
```
YYYY-MM-DD_简短描述.md
```

### 正文模板

```markdown
# 主题

## 现象
观察到了什么。

## 根因（如有）
为什么会这样。

## 结论 / Action
- 后续建议...
- 是否需要调整权重 / 删除 / 替换...
```

### 约束
- **不写具体因子数值**：只写规律、判断、不写当日具体数字
- **一事一文件**：每个主题独立文件，方便 grep 检索
- **必须写 Action**：记录是为了后续行动，不能只描述现象

## 与 factor-recon-daily 的区别

| | factor-recon-daily | factor-notes |
|---|---|---|
| **受众** | 下一个接手的 agent | 未来的自己或团队成员 |
| **内容** | 当日全景：改了什么、还剩什么、建议顺序 | 单点深入：某个因子的质量判断、某个坑的细节 |
| **格式** | 固定模板，按章节填写 | 自由格式，一事一文件 |
| **触发** | "总结一下"、"写日报" | "记一下"、"这个要注意" |
| **输出** | `/home/agent_daily/YYYYMMDD.md` | `/home/notes/{category}/YYYY-MM-DD_描述.md` |

## 示例

### 例 1：因子质量评估

```markdown
# NR 因子数值过小，稳定性差

## 现象
核对发现 NR 因子的数值范围极小（接近 0），在不同模型 fold 之间波动大，对预测贡献低。

## 根因
NR 依赖 mid_price 的短周期与长周期收益率乘积，而该品种近期波动率较低，导致 RETCO * RETOC 长期接近 0。

## 结论 / Action
- 建议降低 NR 在模型中的使用权重
- 后续若波动率回升，可重新评估
- 考虑用替代因子（如 realized_vol）补充
```

### 例 2：技术债务

```markdown
# _align_and_check 的 ffill 导致跨品种因子异常

## 现象
无夜盘品种（LH）在夜盘被 ffill 填充白天盘旧数据，导致 P_LH_* 交叉因子夜盘非 NaN。

## 根因
`strategies.py` 中 `_align_and_check` 使用 `reindex(method='ffill')`，未区分交易时间与非交易时间。

## 结论 / Action
- 已修复：ffill 后删除非交易时间的行（2026-05-13）
- 后续若增加新品种，需检查其 trade_hours 是否正确配置
```
