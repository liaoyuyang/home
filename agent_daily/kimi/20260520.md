## 今天做了什么（代码变更摘要）

### 修改 1：`_save_min_data` 中 tick 区间修正（online + dev）
- **问题**：交易记录中开仓价/平仓价大量为 NaN
- **根因**：`_save_min_data` 用**当前分钟**的 tick 计算 twap，但策略触发时当前分钟 tick 还不完整（被 `_package_data` 截断到 `raw_time`）
- **修复**：`tick_mask` 从当前分钟改为**上一分钟**（`current_minute - 1min`），因为分钟线是右端点（`ts=ceil('1min')`），`ts=21:19:00` 对应的数据区间是 `21:18:00.001~21:19:00.000`
- **文件位置**：
  - `/home/online/dce农/strategies.py`（生产环境）
  - `/home/strategy_PAMY_dev/strategies.py`（开发环境）

### 修改 2：`second >= 6` 改为 `second > 5`（online + dev）
- **问题**：05.05 秒这种时间被错误排除/包含
- **根因**："前五秒不要"的语义应该是 `second > 5`，不是 `second >= 6`
- **修复**：所有排除前5秒的逻辑统一改为 `second > 5`
- **文件位置**：
  - `/home/online/dce农/strategies.py`
  - `/home/strategy_PAMY_dev/strategies.py`

### 修改 3：`calc_recent_data.py` 中 `last_twap` 排除前5秒（online + dev）
- **问题**：盘后补算行情数据时，`last_twap` 是整分钟所有 tick 的均值，没有排除前5秒
- **根因**：`df['last_twap'] = df['last_price']` 直接赋值，未做过滤
- **修复**：`df['last_twap'] = np.where(df['second'] > 5, df['last_price'], np.nan)`，配合 `agg_dict` 中 `'last_twap': 'mean'` 自动忽略 NaN
- **文件位置**：
  - `/home/strategy_online/strategy_PAMY_dce/calc_recent_data.py`
  - `/home/strategy_PAMY_dev/calc_recent_data.py`
  - `/home/strategy_PAMY_dev/data_function.py`
