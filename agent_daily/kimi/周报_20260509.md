# 周报 2026.05.05–05.08

## 本周重点工作

### 1. Realtime 执行引擎性能优化
**问题**：策略编排器使用 ThreadPoolExecutor（多线程），受 Python GIL 限制，CPU 多核无法充分利用，策略并行计算耗时过长。
**解决**：将 `orchestrator.py` 重构为 ProcessPoolExecutor（spawn 模式），突破 GIL 瓶颈；引入子进程级策略对象缓存，避免每轮重复加载模型，显著降低单轮计算耗时。

### 2. 因子计算入口稳定性修复
**问题**：`calc_factors_dce农.py` 通过字符串 hack 推算前一年合约来获取主力日期，因历史合约缺失频繁触发 `IndexError`，导致批量因子计算中断。
**解决**：重构 `_get_df_symbol` 逻辑，直接读取 `main_instrument.csv` 自带的 `end_date` / `instrument_next` 字段，彻底消除合约缺失导致的崩溃，降低新合约上线维护成本。

### 3. Realtime 与 Research 数据对齐
**问题**：Realtime 行情预处理存在 11:30 重复 tick，导致 volume/turnover 被重复计算，与研究环境不一致。
**解决**：在 `calc_recent_data.py` 增加 `drop_duplicates` 去重；以 P 合约为样本完成行情数据对齐验证（OHLC / 持仓已做到 0 diff）。

4. 验证预测值一致性，推动 Realtime 环境具备上线条件。
