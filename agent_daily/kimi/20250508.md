## 已知陷阱（别踩）

- **Trade-date 规则**：夜盘 21:00-23:00 归属 **下一个** trade_date，notebook 里已经处理了
- **Dedup 顺序**：volume/turnover 必须先 diff 再 dedup，顺序反了 boundary 会错
- **Boundary tick**：tick-level factor（如 LVC）需要 08:59、09:00、13:30 这些边界 tick 来保持 rolling window 连续，不能提前过滤 `is_trading=False`
- **Zigzag 类因子**：需要至少 5 日 + buffer 的历史，`_truncate_by_trade_date` 不能截太短
- **Session 过滤**：统计 fail ratio 时要先 `filter_active_trading` 剔除前后 10min，否则开盘噪声会拉高 diff
