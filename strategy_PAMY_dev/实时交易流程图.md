# 实时交易流程图

## 一、架构总览

```mermaid
flowchart TD
    subgraph 行情接入层
        ZMQ[交易所ZMQ行情]
        SW[sql_writer_dce.py]
        DB[(tick_data.db)]
        ZMQ -->|tick| SW
        SW -->|INSERT| DB
    end

    subgraph 数据服务层
        DS[DataService<br/>后台线程]
        DS -->|initialize| CSV[加载历史CSV]
        DS -->|run| DB
        DS -->|每0.5s轮询| REF[_refresh_instrument]
        REF -->|读DB尾部| TC[tick_cache]
        REF -->|聚合| MC[min_cache]
        DS -->|新分钟触发| PKG[_package_data]
        PKG --> Q[Queue]
    end

    subgraph 策略编排层
        ORCH[Orchestrator<br/>主进程]
        Q --> ORCH
        ORCH -->|预检查+重试| CHECK[数据齐备性检查]
        CHECK -->|不齐| WAIT[等待0.5s<br/>重新打包]
        WAIT --> CHECK
        CHECK -->|齐备| TPE[ThreadPoolExecutor<br/>4策略并行]
    end

    subgraph 策略执行层
        ST1[Strategy C]
        ST2[Strategy B]
        ST3[Strategy CS]
        ST4[Strategy LH]
        TPE --> ST1
        TPE --> ST2
        TPE --> ST3
        TPE --> ST4
    end

    subgraph 信号输出层
        ST1 --> Z1[ZMQ 44229]
        ST2 --> Z2[ZMQ 44230]
        ST3 --> Z3[ZMQ 44231]
        ST4 --> Z4[ZMQ 44232]
    end
```

## 二、DataService 内部流程

```mermaid
flowchart TD
    A[启动] --> B[initialize]
    B --> B1[加载 files/*_min.csv<br/>历史分钟线]
    B --> B2[全量读取 tick_data.db<br/>已有tick]
    B2 --> B3[aggregate_ticks<br/>聚合为分钟线]
    B3 --> B4[update_concat<br/>合并历史+实时]
    B4 --> C[min_cache]
    B1 --> C
    
    C --> D[start<br/>后台线程]
    D --> E[run<br/>while循环0.5s]
    E --> F[_refresh_instrument<br/>逐个合约]
    F --> G[read_table limit=50000<br/>读DB尾部]
    G --> H{有新tick?}
    H -->|否| E
    H -->|是| I[更新tick_cache]
    I --> J[aggregate_ticks]
    J --> K[update_concat<br/>更新min_cache]
    
    K --> L[_get_latest_datetime<br/>取最新tick时间]
    L --> M{分钟变化?}
    M -->|否| E
    M -->|是| N[time_recently<br/>截断到整分钟]
    N --> O[sleep 0.5s]
    O --> P[_package_data]
    P --> P1[tick: tail 1000]
    P --> P2[min: 完整min_cache]
    P --> P3[timestamp]
    P --> Q[Queue.put]
    Q --> E
```

## 三、Orchestrator 主循环

```mermaid
flowchart TD
    A[main] --> B[加载配置]
    B --> C[DataService初始化+启动]
    C --> D[初始化策略]
    D --> D1[加载5折LGB模型]
    D --> D2[_load_pred_cache<br/>加载历史parquet缓存]
    D2 -->|无缓存| D3[_init_history<br/>load_fac_df_old计算历史因子]
    D3 --> D4[模型预测pred_df]
    D4 --> D5[保存fac_old.csv<br/>pred_df.csv]
    D2 -->|有缓存| E[主循环while]
    D5 --> E
    
    E --> F[Queue.get阻塞等待]
    F --> G[收到pkg]
    G --> H[time_recently]
    
    H --> I[数据齐备性检查]
    I --> J{所有策略<br/>_align_and_check通过?}
    J -->|否| K[等待0.5s]
    K --> L[data_service._package_data<br/>重新打包]
    L --> I
    J -->|是| M[ThreadPoolExecutor<br/>并行提交策略]
    
    M --> N1[run_strategy C]
    M --> N2[run_strategy B]
    M --> N3[run_strategy CS]
    M --> N4[run_strategy LH]
    
    N1 --> O1[返回结果]
    N2 --> O2[返回结果]
    N3 --> O3[返回结果]
    N4 --> O4[返回结果]
    
    O1 --> P[汇总打印表格]
    O2 --> P
    O3 --> P
    O4 --> P
    
    P --> Q{time_recently==15:00?}
    Q -->|否| E
    Q -->|是| R[_save_pred_cache<br/>按交易日拆分保存parquet]
    R --> S[退出]
```

## 四、单策略 on_new_minute 内部流程

```mermaid
flowchart TD
    A[on_new_minute] --> B[_extract_data<br/>从pkg取tick+min]
    B --> C[_align_and_check<br/>以main symbol为基准]
    C --> C1{其他品种<br/>长度>main?}
    C1 -->|是| C2[截断到main_len]
    C1 -->|否| C3{其他品种<br/>长度<main?}
    C3 -->|交易时段内| C4[返回False<br/>数据不齐]
    C3 -->|非交易时段| C5[允许缺失]
    C2 --> D
    C5 --> D
    
    D[Factor_generator] --> D1[_load_data_tick]
    D --> D2[_load_data_min]
    D2 --> D3[_tick_indexed<br/>预缓存set_index]
    D2 --> D4[valid_index<br/>实时模式截断到最后1个点]
    D --> D5[load_df_names<br/>预缓存_symbol_data_indexed]
    
    D5 --> E[generate_factor<br/>单线程顺序执行]
    E --> E1[parallel_factor_compute<br/>已改为单线程]
    E1 --> E2[300个因子逐个eval]
    E2 --> E3[组装fac_df]
    
    E3 --> F[predict]
    F --> F1[5折模型分别预测]
    F --> F2[weighted加权]
    F --> F3[拼接到pred_df]
    
    F3 --> G[run_345]
    G --> G1{交易时段内?}
    G1 -->|是| G2[计算阈值/分位数]
    G2 --> G3[开仓/平仓/等待信号]
    G1 -->|否| G4[强制平仓信号]
    
    G3 --> H[_save_results]
    G4 --> H
    H --> H1[保存predictions_*.csv]
    H --> H2[保存factors_*.csv]
    H --> H3[保存trading_status_*.json]
    
    G3 --> I[ZMQPublisher.publish]
    G4 --> I
    I --> J[返回信号+耗时]
```

## 五、关键时序与耗时（优化后预估）

| 阶段 | 耗时 | 说明 |
|------|------|------|
| DataService 轮询 | 0.5s | 固定间隔 |
| _package_data | ~0.001s | 内存拷贝 |
| Orchestrator 重试 | 0~1.5s | 仅在不齐时发生 |
| extract | ~0.003s | 字典提取 |
| align | ~0.001s | 长度检查+截断 |
| fac_init | ~0.05s | Factor_generator初始化 |
| **generate** | **~3.0s** | 单线程300因子（优化后） |
| predict | ~0.02s | LGB预测 |
| run345 | ~0.01s | 阈值判断 |
| save | ~0.01s | 写CSV |
| **单策略总耗时** | **~3.1s** | 由generate主导 |
| **4策略并行总耗时** | **~3.1s** | ThreadPoolExecutor并行 |

## 六、数据流关键设计

```
┌─────────────┐     tick      ┌─────────────┐     Queue      ┌─────────────┐
│ sql_writer  │ ─────────────→│ DataService │ ─────────────→│ Orchestrator│
│  (写DB)     │               │ (读DB+缓存)  │              │ (调度策略)   │
└─────────────┘               └─────────────┘              └──────┬──────┘
                                                                  │
                              ┌───────────────────────────────────┘
                              │ 每个策略独立线程
                              ▼
                    ┌─────────────────┐
                    │  Strategy X     │
                    │  ├─ tick_data (tail 1000)
                    │  ├─ min_data  (历史+实时)
                    │  ├─ Factor_generator
                    │  ├─ 300因子计算
                    │  ├─ 5折LGB预测
                    │  ├─ 阈值判断(run345)
                    │  └─ ZMQ发布信号
                    └─────────────────┘
```
