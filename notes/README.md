# Notes — 工程笔记与因子日记

## 这是什么

`/home/notes` 是策略开发团队的**工程笔记目录**，用于记录因子分析、策略调优、代码重构、踩坑备忘等碎片化但高价值的信息。

与 `factor-recon-daily`（给下一个 agent 的状态简报）不同，`notes` 更侧重于：
- **因子质量评估**：某个因子是否稳定、是否值得保留、权重如何调整
- **技术债务记录**：代码里埋了哪些坑、后续要重构什么
- **实验结论**：试了什么方案、效果怎样、为什么放弃
- **快速备忘**：不需要写完整日报，但需要记下来防止遗忘

## 文件组织

```
/home/notes/
├── README.md           # 本文件
├── factor/             # 因子相关笔记
│   ├── 2026-05-13_NR因子数值过小.md
│   └── ...
├── infra/              # 基础设施、数据流水线
├── model/              # 模型训练、调参
└── misc/               # 其他杂项
```

## 写作规范

1. **一事一文件**：每个主题单独一个 markdown 文件，方便检索
2. **文件名格式**：`YYYY-MM-DD_简短描述.md`
3. **必须包含**：
   - **现象**：观察到了什么
   - **根因**（如有）：为什么是这样
   - **结论/Action**：后续怎么做
4. **不写具体因子数值**：写规律、写判断、不写当日的具体数字

## 什么时候写

- 核对因子时发现某个因子长期不稳定
- 调试代码时发现需要后人注意的陷阱
- 做实验（如调整阈值、更换模型）后需要记录结论
- 用户口头交代了后续 TODO，需要固化下来
