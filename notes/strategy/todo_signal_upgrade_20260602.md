# 信号系统升级待办 — 下周执行

## 背景
当前策略输出离散信号为 `{-1, 0, 1}`（空仓/空仓/多仓/空）。
用户提出需要更连续的分档信号 `{-2, -1, 0, 1, 2}`，以便：
- 对强信号加大仓位
- 对弱信号降低仓位或观望
- 提升资金利用效率与夏普

## 具体改造点

### 1. 模型输出改造
- **现状**：LightGBM 预测未来 N 日收益，通过 th1/th2 阈值映射为 -1/0/1
- **目标**：改为 5 档映射
  - `pred >= th1`          →  **2**  (强多)
  - `th2 <= pred < th1`    →  **1**  (弱多)
  - `-th2 < pred < th2`    →  **0**  (观望)
  - `-th1 < pred <= -th2`  →  **-1** (弱空)
  - `pred <= -th1`         →  **-2** (强空)

### 2. 回测层改造 (`backtest_v3` 或 `pipeline.py`)
- `pos` 字段从 `{-1, 0, 1}` 扩展到 `{-2, -1, 0, 1, 2}`
- 仓位权重：可等权（±2 就是 ±2 手）或按信号绝对值加权
- 需要确认交易所/风控对最大持仓的限制

### 3. 参数扫描范围扩展
- 当前：`th1 ∈ [0.8, 0.9], th2 ∈ [0.4, 0.5]`
- 5 档信号后，可能需要额外扫描 `th1/th2` 的更多组合，因为分档更细后对阈值更敏感

### 4. 评估维度增加
- 除现有 `monthly_ret/sharpe/max_dd/win_rate` 外，增加：
  - **信号分布**：各档信号占比（避免某一档过少）
  - **档位命中率**：强信号（±2）的胜率 vs 弱信号（±1）
  - **换手率**：档位切换频率是否过高
